This page tests four approaches to embedding an animated SVG, to verify which survive Mintlify's image optimizer on deployment.

## Approach A — Markdown image reference (BROKEN on deploy)

The optimizer detects `width`/`height` on the SVG, generates imgix srcset URLs, and imgix rasterizes the SVG to PNG/WebP — stripping all SMIL animations.

![Animated SVG via markdown](/images/test-animation.svg)

---

## Approach B — Inline SVG markup (BROKEN on deploy)

MDX strips all SVG children during bundling — the `<svg>` tag renders empty.

<div>
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 200 200" width="200" height="200">
  <circle cx="100" cy="100" r="40" fill="#927746">
    <animate attributeName="r" values="40;70;40" dur="2s" repeatCount="indefinite" />
    <animate attributeName="opacity" values="1;0.3;1" dur="2s" repeatCount="indefinite" />
  </circle>
  <text x="100" y="170" textAnchor="middle" fontSize="14" fill="#e0d390">Pulsing circle</text>
</svg>
</div>

---

## Approach C — HTML `<img>` tag (BROKEN on deploy)

Mintlify also processes `<img>` tags through imgix — same rasterization problem as A.

<img src="/images/test-animation.svg" width="200" height="200" alt="Animated SVG via img tag" />

---

## Approach D — `<object>` tag (testing)

An `<object>` tag loads the SVG as a full document in its own browsing context. Imgix doesn't touch it, and MDX doesn't strip its contents.

<object type="image/svg+xml" data="/images/test-animation.svg" width="200" height="200">SVG not supported</object>

| B — Inline SVG | ✅ animated | ❌ MDX strips children |
| C — `<img>` tag | ✅ animated | ❌ imgix rasterizes |
| D — `<object>` tag | ? | ? |
